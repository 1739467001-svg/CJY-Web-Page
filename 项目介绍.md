# 项目作品集 · 小龙虾生态 🦞
### Works of CJY — the OpenClaw / Lobster Ecosystem

> 陈俊烨 (CJY) · 浙江工商大学 信电人工智能学院 · Vibe Coder
> 一条主线：**把每一个真实需求，养成一只能干活的智能体** —— 再给它一个看得见的数字孪生世界。

我的作品大多生长在同一套方法论上 ——「小龙虾 🦞」：从校园与产业里的**真实场景**出发，
先用 **AI 智能体（OpenClaw Skills）** 接管繁琐流程，再用 **数字孪生（Digital Twin）**
把系统的运行状态三维、实时地呈现出来。所以它们共享同一套底层品牌：智能体叫「虾 / Claw」，
框架叫 **OpenClaw（OC）**，整体叫 **小龙虾生态**。

下面按「生产系统 / AI 智能体 / 数字孪生」三类介绍。文中所有数字均来自各系统线上界面的真实展示。

## 🛡️ 生产系统 / Production System

### 01 · 研究生院督导管理系统
**浙江工商大学研究生院 · 教学督导评价平台**

专业、高效的研究生课程督导评价平台，助力教学质量持续提升。教工凭工号登录，
按角色进入各自的工作台，把"听课—评价—统计—反馈"的督导全流程搬到线上。

- **覆盖规模**：1431 门课程 · 22 个学院 · 588 位授课教师
- **四类角色**：督导专家 · 督导组长 · 学院教学秘书 · 研究生院主管，多角色分权协作
- **核心能力**：课程督导 · 评价管理 · 数据统计 · 多角色协作
- **定位**：面向研究生院的**真实在用生产系统**（非演示）

## 🦞 AI 智能体 / AI Agent

### 02 · 会议室预约虾（OpenClaw Skills）
**用自然语言一句话搞定高校会议室预约**

OPC 极限挑战赛 · 赛道 B「AI 合伙人」参赛作品（汇报人 CJY 陈俊烨，诸葛斌蒋献团队，
"未来课堂"探索方向）。把"登录系统→找房间→查空闲→填申请→提交审批"的传统 3–5 分钟
流程，压缩成对智能体说一句话、**约 10 秒自动完成**。

- **效率**：传统 3–5 分钟 → AI 10 秒，**提效约 30×**
- **真实数据**：基于浙江工商大学信电学院 **1022 条真实预约数据**
- **四项能力**：
  - 意图解析 —— 相对日期、模糊时段、中英数字混合都能听懂
  - 冲突检测 —— 实时查询排班数据，自动避让冲突
  - 智能推荐 —— 容量最优匹配、按设备需求筛选
  - 容错处理 —— 缺信息会追问，撞车自动给备选
- **技术栈**：OpenClaw Skills 标准交付 · SenseAudio 语音 · 钉钉/飞书/企微/QQ 接入 · 阿里云服务器

🔗 https://meetroomshrimp-gvfhrxz8.manus.space

## 🌐 数字孪生 / Digital Twins

### 03 · 会议室预约虾 · 3D 数字孪生（Reality-Twin）
**浙江工商大学 · 会议室预约的三维现实镜像**

会议室预约虾的"数字孪生"一面：把每一间会议室 1:1 建成可交互的三维空间，
实时映射占用状态，并能在 3D 场景里直接预览、预约某个时段。是智能体之外、
给人看的"现实镜像"。

- **实时态势**：今日预约 26 条 · 当前占用 2/6 · 整体利用率 43%
- **空间孪生**：6 间会议室三维建模，按楼层组织（如 信电楼 317 / 318 / 330 / 513 / 529…）
- **时段回放**：08:00–21:00 全天时间轴，空闲 / 已占 / 我的预约 一目了然，可在 3D 中预览
- 校训：诚毅勤朴

> 与 **会议室预约虾** 同源：智能体负责"动手办事"，Reality-Twin 负责"看得见"。

🔗 https://zjgsu.vercel.app/

### 04 · CARGO CLAW（智慧港口数字孪生 v3.0）
**五个智能体自治协同的港口「智能体社会」**

一座会自己运转的智慧港口数字孪生：岸桥、AGV、堆场、船舶在三维世界里实时作业，
而幕后是 **OC Agent Society（OpenClaw 智能体社会）**—— 五个智能体相互通信、
协商、达成共识后再下发指令，把集装箱全流程跑通。

- **OC 智能体社会**：箱单 OC（Data）· 堆叠 OC（Lobster）· 安全 OC（Safety）· 调度 OC（Dispatch）· 指令 OC（Execution）
- **协同闭环**：堆叠 OC 提方案 → 安全 OC 校核 CoG 批准 → 调度 OC 达成共识 → 指令 OC 下发执行
- **实时指标**：TEU/日 885 · 岸桥利用率 77.6% · AGV 行程 234 · 安全评分 99.2
- **集装箱全流程**：订舱 → 集港进闸 → 堆场堆存 → 配载计划 → 岸桥装船 → 离港海运 ⟷ 船舶到港 → 岸桥卸船 → 进口堆存 → 海关查验 → 提柜出闸 → 内陆交付

🔗 https://cargo-claw.vercel.app

### 05 · 天府TWIN（成都天府国际机场孪生）
**机场调度算法 A/B 影子仿真**

成都天府国际机场（TFU）的数字孪生，核心是一台**算法对比沙盘**：在同一份时刻表上，
让「基线 FCFS」与「智能调度」两套算法**同时影子仿真**，三维呈现进近、着陆、滑行、
停靠全过程，并实时给出对比指标。

- **A/B 影子仿真**：FCFS 基线 vs 智能调度，同一时刻表并行对比
- **对比指标**：准点率 100% / 100% · 平均延误 8s / 8s · 平均滑入 0s / 49s · 跑道占用率 02L 95% / 02R 81%
- **可玩性**：1× / 10× / 60× 倍速 · 自动天气 · 自动昼夜 · 塔台事件流 · 多机位视角
- **实时航班**：进近 / 着陆滑跑 / 滑行入位 / 停靠登机 多状态可视

🔗 http://aerotwin-tfu.vercel.app/
